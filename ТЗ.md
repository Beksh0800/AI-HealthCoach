Техническое задание на разработку системы «AI-HealthCoach»
1. Название и назначение проекта
Тема: Разработка мобильного приложения для адаптивной физической реабилитации и фитнеса с использованием алгоритмов искусственного интеллекта. Рабочее название: AI-HealthCoach. Назначение: Приложение предназначено для генерации персонализированных планов тренировок для людей с ограничениями по здоровью (проблемы с позвоночником, суставами, восстановление после травм), исключая опасные нагрузки.

2. Цели проекта
Безопасность: Минимизировать риск травм за счет исключения упражнений, противопоказанных пользователю.

Адаптивность: Динамически изменять план тренировки в зависимости от текущего самочувствия пользователя (Daily Check-in).

Доступность: Заменить дорогостоящие консультации реабилитолога автоматизированным ИИ-помощником.

3. Стек технологий и Инструменты
Для реализации выбраны современные технологии, обеспечивающие кроссплатформенность и масштабируемость:

Язык программирования: Dart.

Фреймворк: Flutter (версия 3.x+).

Архитектура приложения: Clean Architecture + Feature-First (разделение на слои Data, Domain, Presentation).

Управление состоянием (State Management): BLoC / Cubit.

Backend & База данных: Google Firebase (Authentication, Firestore Database).

Искусственный интеллект: OpenAI API (модель gpt-4o-mini или gpt-3.5-turbo) для генерации JSON-структур тренировок.

Дизайн: Material Design 3.

4. Функциональные требования (Что система должна делать)
4.1. Модуль «Профиль здоровья» (Onboarding)
Система должна собирать и хранить медицинский анамнез пользователя.

Входные данные: Возраст, вес, уровень активности, список травм (выбор из справочника: «Грыжа L5-S1», «Травма мениска» и т.д.).

Функция: Создание «Карты ограничений» (список запрещенных движений, например: «Исключить осевую нагрузку»).

4.2. Модуль «Ежедневный чек-ин» (Daily Check-in)
Перед каждой тренировкой система должна опрашивать пользователя.

Параметры опроса: Уровень боли (0-10), локализация боли (если есть), уровень усталости, качество сна.

Реакция системы: Если уровень боли > 4, тренировка переключается в режим «ЛФК/Растяжка» или отменяется.

4.3. Модуль «ИИ-Генератор» (Core AI Logic)
Система должна отправлять промпт (запрос) к API и получать ответ в формате JSON.

Логика:

Сбор данных профиля + данных чек-ина.

Формирование системного промпта («Ты реабилитолог...»).

Получение списка упражнений (название, кол-во повторов, ссылка на анимацию, комментарий врача).

Валидация ответа (проверка на ошибки).

4.4. Модуль «Плеер тренировки»
Отображение текущего упражнения (GIF/Видео).

Таймер отдыха.

Функция «Тревожная кнопка» (Pain Tracker): Кнопка «Мне больно» должна мгновенно останавливать таймер и предлагать замену упражнения.

5. Требования к Искусственному Интеллекту
ИИ должен выступать не как чат-бот, а как генератор структурированных данных.

Пример структуры запроса (Prompt Requirement):

"Учитывая, что у пользователя [Грыжа поясничного отдела], СОСТАВЬ тренировку на 15 минут. ИСКЛЮЧИ прыжки и скручивания. Верни ответ ТОЛЬКО в формате JSON".

Ожидаемый формат ответа:

JSON

{
  "workout_focus": "Укрепление кора без осевой нагрузки",
  "exercises": [
    {
      "name": "Ягодичный мостик",
      "reps": "15",
      "doctor_comment": "Держи поясницу нейтральной, не прогибайся сильно вверх."
    }
  ]
}
6. Структура Базы Данных (Firestore)
Для диплома достаточно 3 основных коллекций:

users:

uid (ID)

medical_profile (Map: диагнозы, ограничения)

goals (String)

exercises_library (Справочник упражнений - опционально, либо берем из ИИ):

id

name

video_url

target_muscles

history (История тренировок):

user_id

date

feedback (оценка сложности)

generated_plan (копия того, что дал ИИ)
