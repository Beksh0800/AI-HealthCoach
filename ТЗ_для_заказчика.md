Техническое задание (Функциональные требования)
Проект: Мобильное приложение «AI-HealthCoach»
Дата: 04.02.2026
Статус документа: На согласование
________________________________________
1. Общие сведения о проекте
•	Наименование продукта: AI-HealthCoach (рабочее название).
•	Тип приложения: Персональный ассистент для адаптивных тренировок и реабилитации с поддержкой ИИ.
•	Целевая аудитория: Люди с сидячим образом жизни, хроническими болями (спина, суставы), проходящие восстановление после травм.
•	Целевая платформа: Android (версия 8.0 и выше).
•	Язык интерфейса: Русский.
•	Режим работы: Онлайн (требуется интернет).
________________________________________
2. Основные бизнес-цели
1.	Обеспечить пользователей безопасными тренировками, которые автоматически подстраиваются под диагнозы.
2.	Снизить стоимость реабилитации, заменив "живого" тренера цифровым аналогом.
3.	Повысить удержание пользователей за счет геймификации и умных уведомлений.
________________________________________
3. Детальное описание функций
3.1. Регистрация и «Цифровой паспорт здоровья»
•	Анкета (Сбор данных): Биометрия (вес, рост), Цели, Уровень активности.
•	Медицинский блок: Пользователь выбирает проблемы из списка (грыжа, артроз, сколиоз и др.).
•	Результат: Формирование скрытого «Списка запретов» (например, при грыже блокируются прыжки).
3.2. Ежедневный опрос (Check-in)
Перед каждой тренировкой система уточняет состояние:
•	«Как самочувствие?» (Слайдер).
•	«Где болит сегодня?» (Выбор зоны тела).
•	Логика: При жалобах на боль нагрузка снижается автоматически.
3.3. ИИ-Генератор тренировки (Стратегия внедрения)
Центральная функция генерации плана занятий.
•	Этап 1 (MVP): Использование бесплатных производительных моделей (Google Gemini API) для отработки алгоритмов без бюджета.
•	Этап 2 (Развитие): Возможность подключения платных шлюзов (OpenRouter/GPT-4) для повышения точности при масштабировании.
•	Результат: Уникальный план упражнений, созданный в реальном времени под текущее состояние пользователя.
3.4. Экран выполнения (Плеер тренировки)
•	Визуализация: Видео/GIF упражнения.
•	Умные подсказки: Персонализированный текст (советы по технике с учетом травм).
•	Функция безопасности «Стоп-Боль»: Кнопка на экране, которая мгновенно останавливает тренировку и запрашивает причину дискомфорта для замены упражнения.
3.5. Система удержания и Уведомления
•	Напоминания: Настраиваемые локальные уведомления («Пора размять спину», «Ты не тренировался 2 дня»).
•	Обратная связь: После тренировки пользователь оценивает сложность. Эти данные используются для корректировки следующей сессии.
________________________________________
4. Безопасность и Данные
•	Приватность: Медицинские данные передаются в ИИ в обезличенном виде (без имен и личных ID).
•	Защита: Используются безопасные протоколы передачи данных (HTTPS).
________________________________________
5. Технические стандарты и Инструменты
В данном разделе зафиксирован стек технологий, выбранный для реализации проекта:
•	Мобильная разработка: Фреймворк Flutter (язык Dart). Обеспечивает высокую производительность и плавность анимаций.
•	База данных и Серверная часть: Облачная платформа Google Firebase:
o	Firebase Authentication: Безопасная система входа пользователей.
o	Cloud Firestore: Быстрая NoSQL база данных для хранения профилей и истории.
•	Интеграция ИИ: Взаимодействие через REST API с передачей ответов в структурированном формате JSON.
•	Контроль версий: Код проекта хранится в репозитории GitHub.
